# EasyHR 员工端小程序设计规范

> 版本：v1.0 | 产品：易人事员工端微信小程序 | 目标用户：企业员工（10-50人小微企业）

---

## 1. 设计理念

**核心原则：简单、可信、3步完成**

- 员工打开小程序，第一时间知道要做什么
- 核心操作 ≤ 3 步完成
- 无需专业知识，一看就懂
- 视觉风格：专业企业级 + 轻量亲和

---

## 2. 设计风格

### 2.1 风格名称
**企业 SaaS 移动端 (Enterprise SaaS Mobile)**

| 维度 | 描述 |
|------|------|
| 关键词 | 专业、可信、 Indigo、清晰、亲和 |
| 适用场景 | B2B 后台管理、HR 工具、工资/社保查询 |
| 性能 | ⚡ 极佳 |
| 可访问性 | ✓ WCAG AA |

### 2.2 为什么选这个风格
- 工资/社保涉及敏感个人信息，需要**信任感**和**专业感**
- Indigo 色系在企业级 SaaS 中广泛使用（如钉钉、飞书）
- 简洁清晰，员工（各种年龄段）都能快速上手
- 与 WeUI 原生组件库搭配协调

## 3. 色彩系统

### 3.1 色彩令牌

```css
/* 背景 & 表面 */
--bg:           #F8FAFC   /* 页面背景：冷灰白 */
--surface:      #FFFFFF   /* 卡片/组件表面 */
--surface-2:   #F1F5F9   /* 次级表面（列表奇偶行等） */

/* 文本 */
--text-primary: #0F172A   /* 主标题/重要文字 */
--text-body:    #334155   /* 正文 */
--text-muted:  #64748B   /* 次要文字/占位符 */
--text-disabled: #CBD5E1 /* 禁用状态 */

/* 品牌色 */
--primary:      #4F46E5   /* Indigo-600：主按钮/主强调 */
--primary-light:#EEF2FF   /* Indigo-50：标签背景/hover */
--secondary:    #7C3AED   /* Violet-600：次级强调 */

/* 功能色 */
--success:      #10B981   /* 绿色：成功/已缴 */
--warning:      #F59E0B   /* 橙色：待处理/提醒 */
--danger:       #EF4444   /* 红色：错误/删除 */
--info:         #0EA5E9   /* 蓝色：信息 */

/* 边框 & 分隔 */
--border:       #E2E8F0   /* 默认边框 */
--border-focus: #4F46E5   /* 聚焦边框 */

/* 阴影 */
--shadow-sm:   0 1px 2px rgba(0, 0, 0, 0.05);
--shadow-card: 0 4px 12px rgba(79, 70, 229, 0.08);
--shadow-lg:   0 8px 24px rgba(79, 70, 229, 0.12);
```

### 3.2 色彩使用规则

| 场景 | 颜色 |
|------|------|
| 主按钮 | `--primary` 填充，白色文字 |
| 次级按钮 | 白色填充 + `--primary` 边框 |
| 成功状态（已缴/正常） | `--success` |
| 警告状态（待处理/未缴） | `--warning` |
| 危险操作（删除） | `--danger` |
| 信息展示 | `--info` |
| 卡片背景 | `--surface` |
| 页面背景 | `--bg` |

### 3.3 对比度要求
- 主文本 vs 背景：≥ 4.5:1
- 次要文本 vs 背景：≥ 3:1
- 大文本（≥18pt）≥ 3:1

## 4. 字体系统

### 4.1 字体选择
```css
/* 首选：Noto Sans SC（支持简体中文，Google Fonts） */
@import url('https://fonts.googleapis.com/css2?family=Noto+Sans+SC:wght@300;400;500;600;700&display=swap');

/* 备选：系统默认无衬线 */
font-family: -apple-system, BlinkMacSystemFont, 'PingFang SC', 'Microsoft YaHei', sans-serif;
```

### 4.2 字号层级

| 用途 | 字号 | 字重 | 行高 |
|------|------|------|------|
| 屏幕标题（H1） | 20px | 600 (Semibold) | 1.2 |
| 区块标题（H2） | 16px | 600 (Semibold) | 1.3 |
| 正文 | 15px | 400 (Regular) | 1.5 |
| 标签/辅助 | 13px | 400 (Regular) | 1.4 |
| 小字/说明 | 12px | 400 (Regular) | 1.4 |
| 数字强调 | 24px | 700 (Bold) | 1.1 |

> **注意**：正文字号最小 15px，避免 iOS 自动缩放

## 8. 组件规范

### 8.1 按钮

**主按钮**
```
高度：44px（触摸目标最小值）
背景：--primary（#4F46E5）
文字：白色，15px，600
圆角：999px（药丸形）
字间距：0.5px
```
- Press 状态：scale(0.97)，背景加深 10%
- Loading 状态：显示 spinner，文字变为"处理中..."

**次级按钮**
```
高度：44px
背景：白色
边框：1px solid --primary
文字：--primary，15px，500
圆角：999px
```

**文字按钮**
```
无背景，无边框
文字：--primary，15px，500
下划线：hover 时显示
```

### 8.2 卡片

```
背景：--surface
圆角：12px
阴影：--shadow-md
内边距：16px
```
- 状态：normal / hover(阴影加深) / pressed(scale 0.98)
- 点击时无 Layout Shift

### 8.3 列表项

```
高度：最小 56px（满足 44px 触摸目标）
左对齐图标/头像 + 主标题 + 副标题 + 右箭头
分割线：1px --border，右侧留 16px
```
- 触摸高亮：背景变为 --primary-light

### 8.4 表单输入

```
高度：48px
背景：--surface
边框：1px solid --border
圆角：8px
内边距：0 16px
聚焦：边框变为 --primary，添加 0 0 0 3px rgba(79,70,229,0.1) 外发光
```
- 标签：始终可见，不使用占位符替代标签
- 错误：边框变为 --danger，下方显示错误文案

### 8.5 空状态

```
┌────────────────────────┐
│                        │
│     [插图图标]          │
│                        │
│    暂无工资条记录        │
│   本月工资将在发薪日后    │
│       自动显示          │
│                        │
│    [联系HR] 按钮        │
│                        │
└────────────────────────┘
```
- 图标：使用 SVG，尺寸 64×64
- 说明文字居中，颜色 --text-muted
- 提供操作按钮引导

### 8.6 骨架屏（Loading）

```
背景：--surface
占位块：--primary-light (#EEF2FF)
动画：shimmer 效果，1.5s 循环
```
- 骨架屏替代 spinner，显示实际布局结构
- 加载时间 > 300ms 时显示骨架屏

### 8.7 Toast 提示

```
位置：顶部，距顶部 48px（避开状态栏）
背景：rgba(15, 23, 42, 0.9)（深色毛玻璃）
文字：白色，14px
圆角：8px
自动消失：3s
```
- 不抢占焦点，支持 VoiceOver 播报（aria-live="polite"）

## 9. 动画规范

| 场景 | 时长 | 缓动 |
|------|------|------|
| 按钮按下反馈 | 150ms | ease-out |
| 页面切换 | 250ms | ease-in-out |
| 底部弹窗进入 | 300ms | cubic-bezier(0.32, 0.72, 0, 1) |
| 骨架屏 shimmer | 1.5s | linear，循环 |
| 数字滚动 | 400ms | ease-out |

- 优先使用 `transform` + `opacity`，避免动画导致 Layout Shift
- 支持 `prefers-reduced-motion`：减少/禁用动画
- 退出动画时长 = 进入动画的 60-70%

---

## 10. 可访问性规范

| 规则 | 实现 |
|------|------|
| 触摸目标 | ≥ 44×44px |
| 颜色对比 | 文本 ≥ 4.5:1，大文本 ≥ 3:1 |
| 焦点状态 | 2-4px --primary 外发光 |
| ARIA 标签 | 图标按钮需 `aria-label` |
| 替代文本 | 所有有意义图片需 `alt` |
| 键盘导航 | Tab 顺序与视觉顺序一致 |
| 动态文字 | 支持系统字体缩放，不截断 |
| 减少动画 | 尊重 `prefers-reduced-motion` |

## 13. 技术实现要点

### 13.1 微信小程序框架
- 使用 **WeUI** 作为基础组件库
- 使用 **wx.request** 封装统一请求（含 JWT token 自动注入）
- 使用 **localStorage** 缓存用户信息，减少重复请求
- 分包加载：Tab 页面独立分包，减少首屏体积

### 13.2 关键分包策略
```
pages.json 配置：
{
  "lazyCodeLoading": "requiredComponents",
  "pages": [
    "pages/index/index",      // 主包
    "pages/payslip/index",    // 分包
    "pages/social/index",     // 分包
    "pages/profile/index"     // 分包
  ]
}
```

### 13.3 性能目标
- 首屏渲染 ≤ 1.5s（4G 环境）
- 列表滑动帧率 ≥ 50fps
- API 响应 ≤ 500ms
- 使用骨架屏减少感知等待时间

### 13.4 安全考虑
- 所有页面需登录态访问（JWT token 校验）
- 敏感数据（身份证号、工资）脱敏展示
- 禁止在客户端存储明文敏感信息
- HTTPS 强制，校验证书
