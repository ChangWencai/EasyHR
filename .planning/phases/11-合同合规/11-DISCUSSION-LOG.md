## Area 1: 模板管理界面

| Option | Description | Selected |
|--------|-------------|----------|
| 固定标准模板 | 提供劳动合同/实习协议/兼职合同预置模板，变量预定义，不支持自定义条款，实现最轻量 | ✓ |
| 模块化条款块 | 允许用户增删改合同条款块（段落级别），自由组合；灵活但复杂度高 | |
| 全自定义富文本 | 所见即所得的富文本编辑器，用户自由编辑；灵活度最高但排版一致性难保证 | |

**User's choice:** 固定标准模板
**Notes:** 轻量够用，符合小微企业实际需求；自定义模板增加复杂度且非核心诉求

### Q2: 合同变量范围

| Option | Description | Selected |

| 基本信息变量 | 固定标准模板仅含姓名/身份证/合同期限；薪资等敏感字段不写入正文，避免频繁重签 | ✓ |

## Area 2: 中文 PDF 生成

### Q4: PDF 中文字体方案

| Option | Description | Selected |
|--------|-------------|----------|
| 内嵌开源中文字体 | 直接嵌入 2-4MB 开源字体（思源黑体/阿里巴巴普惠体），不依赖网络，签发合同不受网络影响；Go 二进制略增大 | ✓ |
| 云端字体子集 | PDF 生成时从 CDN 下载字体子集（~200KB）；字体不进仓库，但生成 PDF 时必须联网；网络异常时降级为方框占位符 | |
| 仅英文 PDF | 纯英文内容；不符合劳动合同法要求，不推荐 | |

**User's choice:** 内嵌开源中文字体
**Notes:** 离线可用性是核心需求，合同签署不能因网络问题中断

---

### Q5: PDF 布局

| Option | Description | Selected |
|--------|-------------|----------|
| 固定模板+自动分页 | 固定标准模板，内容长度可控；通过字体大小/行间距微调，确保常用场景下恰好 2-3 页 A4；超出范围允许自动分页 | ✓ |
| 支持横向扩展 | 提供横向排版支持复杂合同条款；横向打印不便 | |

**User's choice:** 固定模板+自动分页
**Notes:** 简洁一致，劳动合同长度通常可控

## Area 3: 签署验证流程

### Q6: 员工签署验证方式

| Option | Description | Selected |
|--------|-------------|----------|
| 手机号+短信验证码 | 员工收到短信后点击链接进入 H5 页面，输入手机号+6位验证码确认签署；无需微信登录/下载APP，对小微企业员工最友好 | ✓ |
| 小程序内验证+签署 | 员工必须先在微信小程序登录并绑定手机号，才可查看和签署合同；安全性更高但需员工完成注册流程 | |

**User's choice:** 手机号+短信验证码
**Notes:** 最轻量，员工无需提前注册，收到短信即可签署；降低员工使用门槛

---

### Q7: 签署链接有效期

| Option | Description | Selected |
|--------|-------------|----------|
| 7天有效期 | 签署链接 7 天有效期；超过后老板重新发起签署，短信重新发送；平衡便捷性和合规性 | ✓ |
| 30天有效期 | 30 天有效期；等待时间过长可能导致合同纠纷风险 | |
| 永久有效 | 永久有效；合同生命周期管理变复杂 | |

**User's choice:** 7天有效期
**Notes:** 合理时效，兼顾便捷和合规

---

### Q8: 未签署提醒时机

| Option | Description | Selected |
|--------|-------------|----------|
| 3天后自动提醒 | 合同发起后第 3 天，若员工未签署，系统通过待办中心给老板发提醒；及时推动签署流程 | ✓ |
| 仅手动催签 | 需要老板手动点击「催签」才发提醒；减少无效通知但依赖老板主动操作 | |
| 双重节点提醒 | 合同到期前 7 天和到期当天各发一次提醒；通知频繁 | |

**User's choice:** 3天后自动提醒
**Notes:** 自动推动签署流程，3天足够员工处理，也不至于太早
