    **2. 扩展 internal/salary/service.go** — 新增方法（如果 smsClient/cryptoCfg 尚未注入，需修改 NewService 签名）：

    - `SendSlip(orgID, userID int64, recordIDs []int64) ([]SlipSendResult, error)`
      - 校验每个 PayrollRecord 状态必须为 confirmed 或 paid
      - 为每个 record 生成 PayrollSlip：
        1. generateSlipToken() 生成 token
        2. 通过 EmployeeProvider 获取员工手机号
        3. 手机号加密存储（PhoneEncrypted）+ SHA-256 哈希索引（PhoneHash）
        4. ExpiresAt = now + 7 days
        5. Status = sent
        6. 创建 PayrollSlip 记录
      - 返回 [{RecordID, EmployeeName, SlipToken, ShareLink}]
      - ShareLink 格式：`/salary/slip/{token}`（前端拼接完整 URL）

    - `GetSlipByToken(token string) (*SlipDetailResponse, error)`
      - 通过 token 查询 PayrollSlip
      - 校验未过期
      - 不需要 JWT 认证（公开端点）
      - 返回工资单明细（员工姓名脱敏、各项金额、发放日期等）
      - 首次查看时更新 status 为 viewed + viewed_at

    - `VerifySlipPhone(token, phone string) error`
      - 校验 PayrollSlip 未过期
      - 校验手机号匹配（通过 PhoneHash）
      - 发送短信验证码（smsClient.SendCode）
      - 不暴露工资数据，仅验证身份

    - `VerifySlipCode(token, phone, code string) (bool, error)`
      - 校验短信验证码
      - 通过后允许查看工资单

    - `SignSlip(token string) error`
      - 校验 PayrollSlip status 为 viewed
      - 更新 status 为 signed + signed_at = now
      - 已签收不可更改
