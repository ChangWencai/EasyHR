           ApprovedBy      *int64         `gorm:"column:approved_by" json:"approved_by"`

       关键方法:
       - `BossResign(ctx, orgID, userID, employeeID int64, req *BossResignRequest) error`:
         1. empRepo.FindByID(orgID, employeeID) — 验证员工存在且 status 不是 resigned
         2. 生成默认交接清单模板（3个分类，per D-15）
         3. 创建 Offboarding{Type: "involuntary", Status: "pending"}
         4. 解析 req.ResignationDate → time.Time
         5. 立即更新 Employee.status = "resigned", resignation_date, resignation_reason（per D-16，老板直接办理立即生效）

       - `EmployeeApplyResign(ctx, orgID, employeeID int64, req *EmployeeResignRequest) error`:
         1. 验证员工存在且 status 不是 resigned
         2. 生成默认交接清单模板
         3. 创建 Offboarding{Type: "voluntary", Status: "pending"}，等待老板审批

       - `ApproveResign(ctx, orgID, approverID, offboardingID int64) error`:
         1. obRepo.FindByID(orgID, offboardingID)
         2. 验证 status == "pending"
         3. 更新 Offboarding{Status: "approved", ApprovedBy: approverID, ApprovedAt: now}
         4. 更新 Employee.status = "resigned", resignation_date, resignation_reason

       - `UpdateChecklist(ctx, orgID, offboardingID int64, items datatypes.JSON) error`:
         1. obRepo.FindByID(orgID, offboardingID)
         2. obRepo.Update(orgID, offboardingID, map{"checklist_items": items})

       - `CompleteOffboarding(ctx, orgID, offboardingID int64) error`:
         1. obRepo.FindByID(orgID, offboardingID)
         2. 验证 status == "approved" 或 "pending"（老板直接办理可能跳过审批）
         3. 更新 Offboarding{Status: "completed", CompletedAt: now}
         4. 触发 onEmployeeResigned(orgID, employeeID)（per D-17）

       - `GetOffboarding(ctx, orgID, offboardingID int64) (*OffboardingDetailResponse, error)`: 返回离职详情含员工姓名

       - `onEmployeeResigned(orgID, employeeID int64)`: 预留事件接口（per D-17）
         ```go
         func (s *OffboardingService) onEmployeeResigned(orgID, employeeID int64) {
             // Phase 3: 社保停缴提醒
             // s.socialInsuranceSvc.CreateStopReminder(orgID, employeeID)

                 zap.Int64("org_id", orgID),
                 zap.Int64("employee_id", employeeID))
         }
         ```

       - `defaultChecklistItems() datatypes.JSON`: 生成默认交接清单（3分类，per D-15）

       路由注册（per RESEARCH.md API 设计）:
       ```go
       func (h *OffboardingHandler) RegisterRoutes(rg *gin.RouterGroup, authMiddleware gin.HandlerFunc) {
           authGroup := rg.Group("")
           authGroup.Use(authMiddleware)

           // 老板办理离职 — OWNER/ADMIN
           authGroup.POST("/employees/:id/resign", middleware.RequireRole("owner", "admin"), h.BossResign)
           // 员工申请离职（H5，需认证）
           authGroup.POST("/employees/:id/resign/apply", h.EmployeeApplyResign)
           // 审批 — OWNER/ADMIN
           authGroup.PUT("/offboardings/:id/approve", middleware.RequireRole("owner", "admin"), h.ApproveResign)
           // 完成交接 — OWNER/ADMIN
           authGroup.PUT("/offboardings/:id/complete", middleware.RequireRole("owner", "admin"), h.CompleteOffboarding)
           // 离职详情 — 所有角色
           authGroup.GET("/offboardings/:id", h.GetOffboarding)
           // 更新交接清单 — OWNER/ADMIN
           authGroup.PUT("/offboardings/:id/checklist", middleware.RequireRole("owner", "admin"), h.UpdateChecklist)
       }
       ```

       Handler 方法实现（参照 internal/user/handler.go 模式）:
       - BossResign: ParseInt(c.Param("id")) + ShouldBindJSON → svc.BossResign

       - ApproveResign: ParseInt(c.Param("id")) → c.GetInt64("user_id") → svc.ApproveResign
       - CompleteOffboarding: ParseInt(c.Param("id")) → svc.CompleteOffboarding
       - GetOffboarding: ParseInt(c.Param("id")) → svc.GetOffboarding → Success
       - UpdateChecklist: ParseInt(c.Param("id")) + ShouldBindJSON → svc.UpdateChecklist
