## Component Inventory

### 1. SIDashboard (4-Card Stats)

**Location:** `frontend/src/views/socialinsurance/SIDashboard.vue` (new)

**Appearance:** Matches `SalaryDashboard.vue` stat card pattern exactly, adapted to SI domain.

| Card | Label | Accent Color | Trend | Notes |
|------|-------|-------------|-------|-------|
| 应缴总额 | 应缴总额（元） | #4F6EF7 (primary) | % month-over-month | company + personal + overdue sum |
| 单位部分 | 单位部分合计（元） | #4F6EF7 | % month-over-month | company amount only |
| 个人部分 | 个人部分合计（元） | #4F6EF7 | % month-over-month | personal amount only |
| 欠缴金额 | 欠缴金额（元） | #FF5630 (danger) | % month-over-month | overdue total |

**Trend logic:** Green up-arrow if positive for revenue metrics (normal = good); red up-arrow for overdue metric (overdue increasing = bad). Neutral "--" if no prior month data. Reference: `SalaryDashboard.vue` `.stat-trend` styles.

**No month picker** — shows current month data by default (per D-SI-05).

**Loading state:** `v-loading` overlay on the card grid.
**Empty state:** 4 cards with value "0.00" and "--" trend.

### 2. Overdue Banner

**Location:** Inline in `SITool.vue` (参保操作 tab only, per D-SI-10)

**Visibility condition:** Show when `overdueItems.length > 0`

**Layout:**
```
[Warning icon] 最大欠缴：张三 北京 2026-03 欠缴 ¥1,234.56
                [下方滚动列表：所有未处理欠缴项...]
```

- Single static banner (no animation — CONTEXT.md deferred animation decision: keep static)
- Banner is dismissible (X button) but re-appears on next page load
- Shows top 1 overdue item as headline
- Below headline: scrollable mini-list of all overdue items (max-height: 80px, overflow-y: auto)
- Each list item: `员工姓名 城市 欠缴月 ¥金额`
- If overdue count > 5: show first 5 + "还有 N 项" collapsed text

**Style:**
```scss
background: #FF563015;
border-left: 3px solid #FF5630;
border-radius: 8px;
padding: 12px 16px;
color: #FF5630;
font-size: 14px;
```

### 3. EnrollDialog (增员弹窗)

**Location:** `frontend/src/components/socialinsurance/EnrollDialog.vue` (new)

**Trigger:** "增员" button in 参保操作 tab

**Fields:**

| Field | Component | Default | Required | Notes |
|-------|-----------|---------|----------|-------|
| 员工姓名 | `el-select` with filterable remote search | — | Yes | Triggers `employeeApi.search(name)` on input, shows matching list |
| 身份证号 | `el-input` (readonly, auto-filled) | — | Auto | Filled when employee selected |
| 起始月份 | `el-date-picker` type="month" | 当月 | Yes | 可选近3个月（SI-06） |
| 缴费城市 | `el-select` or `el-input` | — | Yes | City for SI calculation |
| 社保基数 | `el-input-number` | — | Yes | SI salary base, precision=2 |
| 公积金基数 | `el-input-number` | — | No | HF base, precision=2 |
| 公积金比例 | `el-input-number` | 12% | No | Range 1-24, step 0.5 |

**Layout:** Single form, vertical stacking. Width: 480px. No tab switch — all fields visible in one scrollable form.

**CTAs:** "取消" (text button) + "确认增员" (primary button, #4F6EF7)

**Pre-submit validation:** All required fields filled. SI base > 0. HF base > 0 if ratio set.

**Success:** Closes dialog, refreshes records table, shows success message.

| 公积金封存日期 | `el-date-picker` | — | No | Defaults same as 转出日期 |

**SI-13 Tooltip:**
```
转出生效规则：
• 每月5日前转出 → 当月生效
• 5-25日转出 → 次月生效
• 25日后转出 → 下下月生效
```

Show as `el-tooltip` with `WarningFilled` icon next to 转出日期 field.

**CTAs:** "取消" (text) + "确认减员" (danger button, #FF5630)

## Interaction Patterns

### Dialog Patterns

- All dialogs use `el-dialog` with `destroy-on-close` to reset state
- Form validation via Element Plus `rules` prop on `el-form`
- Error messages inline below field (Element Plus default)
- Submit button loading state during API call

### Search/Filter Patterns

- Employee search: debounce 300ms before triggering API call
- Use `el-select` with `filterable`, `remote`, and `remote-method`

### Data Refresh

- After EnrollDialog success: refresh records table
- After StopDialog success: refresh records table
- After status change: refresh records table
