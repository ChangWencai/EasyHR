## Interaction Patterns

### 薪资数据看板月份切换

- 切换月份后，4 张卡片数据同时更新（并发请求）
- 环比百分比：上月无数据时显示 "--"（Text Secondary）
- 卡片数据加载中显示 el-skeleton 骨架屏

### 普调部门多选预览

- 选择部门后（el-select multiple），立即显示影响人数（实时计算）
- 点击"预览影响"调用后端 API，返回预估月度/年度影响金额
- 预览面板展示影响人数、预估金额、生效月份，确认后提交
- 预览 API 失败时显示错误提示，不阻塞提交

### 个税 Excel 上传流程

- 文件拖拽或点击上传，自动解析 Excel
- 解析成功后展示匹配结果表格（成功行绿色标签，失败行红色标签）
- 部分成功时 el-alert 提示"已成功处理 N 条，M 行无法匹配"
- 全部失败时 el-alert type=error 提示失败原因
- 点击"确认并更新"调用后端 API，更新成功后提示"工资表个税数据已更新，请重新核算"

### 绩效系数滑块交互

- el-slider 拖动时实时显示当前值 "{n}%"
- 支持直接点击滑块轨道跳转
- 保存时仅提交已修改的系数（dirty tracking）
- 默认系数（100%）不变动时前端不标记为 dirty，不提交
- 批量保存前显示 loading，避免重复提交

### 发工资条批量发送

- 点击"发送工资条"后弹出 el-dialog 确认
- 确认后调用后端 asynq 队列接口，立即返回
- 成功后显示 ElMessage.success，可点击"发送记录"查看详情
- 批量发送在后台进行，发送记录实时刷新（WebSocket 或定时轮询）

### 工资表历史数据保护（交互约定）

- confirmed/paid 状态行显示锁定图标（el-icon: Lock），禁止编辑
- 点击锁定行弹出 el-dialog："该月工资已确认锁定，如需修改请输入解锁码"
- 解锁码输入企业主手机验证码
- 解锁后行变为可编辑状态，但需重新核算并再次确认

### 调薪 INSERT ONLY 策略（交互约定）

- 调薪提交后新增调薪记录，不修改历史工资表
- 当月 draft 工资表自动重新核算（前端显示刷新提示）
- confirmed/paid 月份的工资表不受影响（调薪在 effective_month 自然生效）
- 调薪历史通过调薪记录列表查看，支持按员工/部门/时间筛选
