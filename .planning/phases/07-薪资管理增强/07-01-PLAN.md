    employee_id BIGINT NOT NULL,

    created_by BIGINT,
    updated_by BIGINT,
    created_at TIMESTAMP DEFAULT NOW(),
    updated_at TIMESTAMP DEFAULT NOW(),

);

    error_message TEXT,

### Task 1: SalaryDashboard (backend)

**Files:** `internal/salary/dashboard_service.go`, `internal/salary/dashboard_handler.go`, `internal/salary/dashboard_dto.go`

**Action:**
- Create `SalaryDashboardService` in `dashboard_service.go` with `GetDashboard(ctx, orgID, year, month)` method
- Use `errgroup.WithContext` to query 4 indicators concurrently:
  - `SUM(gross_income) FROM payroll_records WHERE year=? AND month=? AND status IN ('calculated','confirmed','paid')` → 应发总额
  - `SUM(net_income) FROM payroll_records WHERE year=? AND month=? AND status IN ('calculated','confirmed','paid')` → 实发总额
  - `SUM(si_deduction) FROM payroll_records WHERE year=? AND month=?` → 社保公积金总额
  - `SUM(tax) FROM payroll_records WHERE year=? AND month=?` → 个税总额
- Calculate month-over-month %: query same indicators for (year, month-1), compute `(curr - prev) / prev * 100`
- Use `shopspring/decimal` for division to avoid float precision
- If prev month has no data, trend = null (display "--" in frontend)
- Return `SalaryDashboardResponse` with 4 StatItem: `{ label, value, trend_percent, trend_direction }`
- Register `GET /api/v1/salary/dashboard` in handler

**Verify:** `grep -n "SalaryDashboardService\|GetDashboard\|dashboard_handler" internal/salary/`
**Done:** Dashboard service returns 4 stats + trend_percent; handler registered at dashboard route

**Action:**

- `CreateAdjustment(orgID, userID int64, req *AdjustmentRequest) error`:
  - Validate `effective_month >= current_month`
  - If `type=individual`: employee_id required, department_id = nil
  - If `type=department`: department_ids required (array), loop each dept to INSERT per employee
  - If `adjust_by=ratio`: convert ratio% to absolute delta from `old_value`
  - Record `old_value` and `new_value` in DECIMAL
  - NEVER UPDATE existing records
- `GetAdjustmentList(orgID int64, filters)` → list with pagination
- Preview endpoint `POST /api/v1/salary/adjustment/preview`: calculate monthly/annual impact for selected employees/departments
- Register routes in handler

**Verify:** `grep -n "SalaryAdjustment\|CreateAdjustment" internal/salary/`
**Done:** Adjustment INSERT works, preview returns employee_count + monthly_impact + annual_impact
