### Task 1: Tax Upload (backend)

**Files:** `internal/salary/tax_upload_service.go`, `internal/salary/tax_upload_handler.go`

**Action:**

**tax_upload_service.go:**
```go
// TaxUploadRow parsed row from Excel
type TaxUploadRow struct {
    Name           string
    TaxAmount      float64   // 个税金额
    Adjustment     float64   // 应补/应退额（正=补，负=退）
    RowNumber      int      // Excel row number
}

// TaxUploadResult 上传结果
type TaxUploadResult struct {
    TotalRows     int
    MatchedCount int
    UnmatchedRows []UnmatchedRow
}

// UnmatchedRow 无法匹配的行
type UnmatchedRow struct {
    RowNumber   int
    Name        string
    Reason      string  // "未找到匹配员工" / "存在多个匹配"
}

// UploadTaxFile 解析 Excel 并返回匹配结果
func (s *Service) UploadTaxFile(orgID, userID int64, year, month int, file []byte) (*TaxUploadResult, error) {
    // 1. Parse Excel with excelize
    rows, err := parseTaxExcel(file)
    if err != nil {
        return nil, fmt.Errorf("解析 Excel 文件失败: %w", err)
    }
    // 2. Match each row: exact name → fuzzy → skip
    // Name column aliases: ["姓名","员工姓名","纳税人姓名"]
    // TaxAmount aliases: ["个税","税额","个人所得税","本期应扣缴税额"]
    // Adjustment aliases: ["应补/应退额","应补退额","应补（退）额","应补","应退"]
    // 3. For matched rows: UPDATE payroll_records SET tax = matched_value WHERE employee_id=?
    // 4. Mark payroll_records status back to 'draft' (if currently 'calculated' or 'confirmed')
    // Use GORM transaction
}
```

**Column name matching (aliases per D-SAL-TAX-01):**
- Name: `["姓名", "员工姓名", "纳税人姓名"]`
- TaxAmount: `["个税", "税额", "个人所得税", "本期应扣缴税额"]`
- Adjustment: `["应补/应退额", "应补退额", "应补（退）额", "应补", "应退"]`

**Matching logic:**
```
1. Exact match: name == employee.Name
2. Fuzzy match: employee.Name contains name (name is substring)
3. If multiple fuzzy matches: reason = "存在多个匹配员工"
4. If no match: reason = "未找到匹配员工"
```

**On confirmation:**
- Batch UPDATE payroll_records SET tax = new_tax WHERE employee_id IN (...)
- UPDATE payroll_records SET status = 'draft' WHERE year=? AND month=? AND status IN ('calculated','confirmed')
- All in GORM transaction

**Register:** `POST /api/v1/salary/tax-upload` in handler

**Verify:** `grep -n "UploadTaxFile\|parseTaxExcel" internal/salary/`
**Done:** Tax Excel upload returns matched/unmatched rows, confirmation updates payroll_records.tax and resets status to draft

### Task 3: asynq Integration + Batch Slip Send

**Files:** `internal/salary/slip_send_task.go`, `internal/salary/slip_send_service.go`, `internal/salary/slip_send_handler.go`, `cmd/server/main.go`

**Action:**

**slip_send_task.go:**
```go
const TypeSlipSend = "salary:slip:send"

type SlipSendPayload struct {
    OrgID       int64
    UserID      int64
    Year        int
    Month       int
    EmployeeIDs []int64  // empty = all employees
    Channel     string   // miniapp/sms/h5
}

func NewTask(TypeSlipSend, &SlipSendPayload) (*asynq.Task, error) { ... }

func EnqueueSlipSend(payload *SlipSendPayload) error {
    task, _ := NewTask(TypeSlipSend, payload)
    return AsynqClient.Enqueue(task, asynq.MaxRetry(3))
}
```

**slip_send_service.go — SendAllSlips:**
```go
func (s *Service) SendAllSlips(orgID, userID int64, year, month int, employeeIDs []int64, channel string) error {
    // Enqueue asynq task (immediate return, HTTP won't timeout)
    return EnqueueSlipSend(&SlipSendPayload{
        OrgID: orgID, UserID: userID, Year: year, Month: month,
        EmployeeIDs: employeeIDs, Channel: channel,
    })
}
```

**slip_send_handler.go:**
- `POST /api/v1/salary/slip/send-all` — batch send (enqueue asynq task, return "发送中")
- `POST /api/v1/salary/slip/send` — single send (immediate, for selected employees)

**main.go — register asynq Worker:**
```go
import "github.com/hibiken/asynq"
import "github.com/redis/go-redis/v9"

redisConn, _ := redis.Dial("tcp", viper.GetString("redis.addr"))
srv := asynq.NewServer(redisConn, asynq.Config{ Concurrency: 10 })
mux := asynq.NewServeMux()
mux.HandleFunc(salary.TypeSlipSend, salary.HandleSlipSendTask)
go srv.Run(mux)
```
- First: `go get github.com/hibiken/asynq@latest`

**HandleSlipSendTask (in slip_send_service.go):**
- Process payload: get employee list (all or specific IDs)
- For each employee: find PayrollRecord for year/month with status confirmed/paid
- If PayrollSlip already exists: update status, log, continue
- Create PayrollSlip with token (reuse existing pattern)
- Write SalarySlipSendLog (channel, status=sent/sending/failed)
- Send via channel priority: try miniapp first, fall back to sms, then h5
