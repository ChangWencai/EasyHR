  <action>
    创建 pkg/jwt/jwt.go:

    - 定义 Claims struct (per D-06):
      ```go
      type Claims struct {
          UserID int64  `json:"user_id"`
          OrgID  int64  `json:"org_id"`
          Role   string `json:"role"`
          jwt.RegisteredClaims
      }
      ```

    - `func GenerateToken(userID, orgID int64, role, secret string, ttl time.Duration) (string, error)`:
      - claims := Claims{UserID: userID, OrgID: orgID, Role: role, RegisteredClaims: jwt.RegisteredClaims{...}}
      - RegisteredClaims 包含: ExpiresAt(jwt.NewNumericDate(time.Now().Add(ttl))), IssuedAt(jwt.NewNumericDate(time.Now())), ID(uuid.New().String()), Issuer("easyhr")
      - token := jwt.NewWithClaims(jwt.SigningMethodHS256, claims)
      - return token.SignedString([]byte(secret))

    - `func ParseToken(tokenStr, secret string) (*Claims, error)`:
      - token, err := jwt.ParseWithClaims(tokenStr, &Claims{}, func(t *jwt.Token) (interface{}, error) { return []byte(secret), nil })
      - 校验 SigningMethod 是 HS256（防止 algorithm confusion attack）
      - if claims, ok := token.Claims.(*Claims); ok && token.Valid { return claims, nil }
      - return nil, err

    - `func GenerateRefreshToken(userID, orgID int64, role, secret string, ttl time.Duration) (string, error)`:
      - 与 GenerateToken 相同结构，ttl 由调用者传入 720h (30天 per D-06)
      - jti 使用不同的 UUID

    - `func RefreshAccessToken(refreshTokenStr, secret string, accessTTL time.Duration) (string, error)`:
      - claims, err := ParseToken(refreshTokenStr, secret) -- 验证 refresh token 有效
      - if err != nil { return "", err }
      - 使用 claims 中的 UserID/OrgID/Role 生成新 Access Token
      - return GenerateToken(claims.UserID, claims.OrgID, claims.Role, secret, accessTTL)

    - 依赖: github.com/golang-jwt/jwt/v5, github.com/google/uuid

      - TestParseToken_InvalidSignature: 用 secretA 生成 token -> 用 secretB 解析 -> 验证返回错误
      - TestGenerateRefreshToken_Success: 验证返回非空 token
      - TestRefreshAccessToken_ValidRefreshToken: 生成 refresh token -> 刷新 -> 验证新 token 有效
      - TestGenerateToken_UniqueJTI: 连续生成两个 token -> 解析 -> 验证 jti 不同
