       SendCode 实现 per D-07：
       - 检查 Redis `sms:limit:{phone}` 是否存在（60秒限制），存在则返回限流错误
       - 生成 6 位随机数字验证码
       - 调用 SMS 客户端发送
       - 存储到 Redis `sms:code:{phone}`，TTL 5 分钟
       - 设置限流 key `sms:limit:{phone}`，TTL 60 秒

       Login 实现 per D-05/D-08：
       - 从 Redis 获取验证码 `sms:code:{phone}`
       - 不存在 → 返回"验证码已过期"
       - 递增错误计数 `sms:errors:{phone}`，检查是否 >= 5 → 删除验证码 key，要求重新发送 per RESEARCH Pitfall 2
       - 验证码不匹配 → 返回错误
       - 验证码匹配 → 删除验证码 key + 错误计数 key
       - 用 phone 的 SHA-256 哈希查找用户
       - 用户不存在 → 自动注册：创建 Organization（status=inactive）+ 创建 User（role=owner），返回 token（onboarding_required=true）per D-08
       - 用户存在 → 返回 token（onboarding_required=false）
       - 生成 access token（7天）+ refresh token（30天）

       RefreshToken 实现 per D-28：
       - 调用 `jwt.RefreshTokens()`，传入 blacklistFunc 将旧 jti 加入 Redis（TTL = 剩余有效期）

       Logout 实现 per D-29：
       - 将当前 access token 的 jti 加入 Redis 黑名单 `token:blacklist:{jti}`，TTL = token 剩余有效期
       - 如果提供了 refresh token，同样加入黑名单

       CompleteOnboarding 实现：
       - 更新 Organization 的 name/credit_code/city/contact_name/contact_phone
       - 设置 Organization status = active
       - 加密存储 contact_phone（AES + SHA-256 双列）

       - 只有 OWNER/ADMIN 可创建（RBAC 在 Handler 层通过中间件检查）

       - 创建用户，加密手机号（AES + SHA-256 双列）
       - 角色只能是 admin 或 member per D-12（OWNER 唯一 per D-14）

       // RequireRole 检查当前用户角色是否在允许列表中
       func RequireRole(roles ...string) gin.HandlerFunc {
           return func(c *gin.Context) {
               userRole := c.GetString("role")
               for _, r := range roles {
                   if strings.EqualFold(userRole, r) {
                       c.Next()
                       return
                   }
               }
               response.Forbidden(c, "权限不足")
               c.Abort()
           }
