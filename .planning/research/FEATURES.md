# Feature Landscape

**Domain:** 小微企业人事管理系统 (Micro/Small Business HR Management)
**Researched:** 2026-04-06
**Confidence:** MEDIUM (WebSearch/WebReader unavailable; based on PRD analysis, training data knowledge of Chinese HR SaaS market, and competitor product understanding)

## Competitive Landscape

### Direct & Adjacent Competitors

| Competitor | Target | Core Strength | Core Weakness for Target User |
|------------|--------|---------------|-------------------------------|
| 钉钉 (DingTalk) | 中大型企业为主 | 协同办公生态，免费基础功能 | HR功能分散在多个子应用中，配置复杂，小老板难上手 |
| 飞书 (Feishu/Lark) | 中大型企业为主 | 协同+文档+OKR一体化 | 人事模块偏中大型企业设计，社保/个税深度不足 |
| 2号人事部 | 中小企业 | 人事全流程覆盖，社保专业度高 | 功能较重，学习成本不低，财务记账缺失 |
| 蚂蚁HR | 中小企业 | 社保公积金专精，城市覆盖广 | 偏社保代办服务，非一站式，工资/个税/财务弱 |
| 点米社保通 | 中小企业 | 社保政策库全，更新及时 | 聚焦社保，非全流程人事管理工具 |
| 金蝶/用友 | 中大型企业 | 财务+HR全模块成熟 | 过于庞大复杂，价格高，小微企业用不起 |
| 代理记账公司 | 小微企业 | 全托管服务 | 成本高(每月几百元)，数据不在自己手中 |

### Market Gap Analysis

**明确的市场空白：** 没有产品同时做到"轻量化HR全流程" + "基础财务记账" + "3步完成核心操作" + "完全免费"。这正是 EasyHR 的定位。

## Table Stakes

用户期望的标配功能。缺失则产品感觉不完整，用户会流失。

### 核心人事管理

| Feature | Why Expected | Complexity | Priority | Notes |
|---------|-------------|------------|----------|-------|
| 员工入职登记 | 所有HR系统的基础功能，用户第一触点 | Low | P0 | 老板创建邀请 -> 员工填写 -> 自动建档，必须3步内完成 |
| 员工离职办理 | 人事生命周期必备环节 | Low | P0 | 自动生成交接清单，触发后续社保停缴等联动 |
| 员工档案管理 | 所有HR信息的汇聚点 | Low | P0 | 搜索、查看、导出Excel是基本期望 |
| 劳动合同管理 | 《劳动合同法》合规要求 | Med | P0 | V1.0 PDF模板+手动签署是可接受的降级方案 |

### 社保管理

| Feature | Why Expected | Complexity | Priority | Notes |
|---------|-------------|------------|----------|-------|
| 社保参保基数自动匹配 | 这是用户最大痛点，不懂社保政策 | High | P0 | 核心差异化功能，自建30+城市政策库是V1.0正确选择 |
| 社保缴费提醒 | 老板容易忘记缴费时间 | Low | P0 | 到期前3天提醒是标配做法 |
| 社保变更联动 | 薪资变动必须同步社保 | Med | P0 | 岗位/薪资变动自动触发调整提醒 |

### 工资管理

| Feature | Why Expected | Complexity | Priority | Notes |
|---------|-------------|------------|----------|-------|
| 薪资结构自定义 | 每个企业工资构成不同 | Med | P0 | 基本工资+绩效+补贴+扣款，支持灵活配置 |
| 一键工资核算 | 替代Excel手动计算 | Med | P0 | 核心痛点，必须准确 |
| 电子工资单推送 | 员工需要看到工资明细 | Med | P0 | 推送到员工微信小程序 |
| 工资发放记录 | 财务对账需要 | Low | P1 | 记录每月发放状态/金额/方式 |
| 导出工资条Excel | 财务和老板的习惯 | Low | P0 | 导出功能是所有竞品的标配 |

### 个税申报

| Feature | Why Expected | Complexity | Priority | Notes |
|---------|-------------|------------|----------|-------|
| 个税自动计算 | 个税计算复杂，用户不懂专项附加扣除 | High | P0 | 基于工资数据自动匹配扣除项，精准计算 |
| 个税申报提醒 | 容易错过申报截止日 | Low | P0 | 截止前3天提醒 |
| 个税申报表生成 | 减少手动填写错误 | Med | P0 | V1.0生成申报表供手动提交，符合用户预期 |

### 基础设置

| Feature | Why Expected | Complexity | Priority | Notes |
|---------|-------------|------------|----------|-------|
| 企业信息设置 | 首次使用引导 | Low | P0 | 同步至各模块减少重复填写 |
| 多子账号权限 | 老板可能让兼职人事操作 | Med | P1 | OWNER/ADMIN/MEMBER三级RBAC |
| 操作日志 | 审计和追溯需要 | Low | P1 | 谁在什么时间做了什么操作 |

### 员工端

| Feature | Why Expected | Complexity | Priority | Notes |
|---------|-------------|------------|----------|-------|
| 查看工资条 | 员工最关心的功能 | Low | P0 | 微信小程序端 |
| 查看合同/社保记录 | 员工权益知情 | Low | P1 | 信息透明 |
| 费用报销提交 | 员工日常需求 | Med | P1 | 上传票据照片，填写金额 |

## Differentiators

让产品脱颖而出的功能。用户不一定会期望，但会高度认可。

### 体验差异化（最高优先级）

| Feature | Value Proposition | Complexity | Notes |
|---------|-------------------|------------|-------|
| 核心操作3步完成 | 老板无耐心学习复杂流程，这是与钉钉/飞书的核心区别 | Med | 贯穿所有功能的UX原则，而非单独功能。每个核心操作必须验证 <= 3步 |
| 零学习成本上手 | 目标用户无HR专业知识，打开APP就知道做什么 | Med | 智能引导、默认值预填、上下文提示 |
| 手机号验证码一键登录 | 降低注册门槛 | Low | 中国用户习惯，无需记密码 |

### HR+财务一站式

| Feature | Value Proposition | Complexity | Notes |
|---------|-------------------|------------|-------|
| 财务记账集成 | 市面上没有产品同时做好HR+记账，小微企业通常需要两个系统 | High | V1.0手动凭证+发票登记+费用报销+账簿+报表，这是关键差异化 |
| 费用报销闭环 | 员工小程序提交 -> 老板审批 -> 自动生成费用凭证 | Med | HR和财务的联动点，竞品少见 |
| 工资-个税-社保联动 | 修改工资自动联动个税计算和社保基数调整 | High | 全流程数据打通，不需要重复录入 |

### 商业模式差异化

| Feature | Value Proposition | Complexity | Notes |
|---------|-------------------|------------|-------|
| V1.0-V2.0完全免费 | 目标用户对成本极度敏感，钉钉免费但HR模块不全 | Low | 免费是获客利器，用增值服务变现 |

### 技术差异化

| Feature | Value Proposition | Complexity | Notes |
|---------|-------------------|------------|-------|
| 降级方案设计 | 核心业务不依赖第三方接口，断网也能用 | Med | 电子签->手动签署，社保->手动办理，体现可靠性 |
| 离线友好 | 信号不好的地方也能完成操作 | Med | V1.0不强制要求实时在线 |

## Anti-Features

**明确不建设的功能。** 小微企业不需要，建设会分散资源和增加复杂度。

### V1.0 明确排除

| Anti-Feature | Why Avoid | What to Do Instead |
|--------------|-----------|-------------------|
| 考勤管理(打卡/定位) | 需对接硬件设备，复杂度极高；10-50人小企业多靠信任管理 | V2.0再考虑，V1.0支持导入考勤表辅助工资核算 |
| 电子签API自动签署 | 第三方依赖成本高(按份收费)，V1.0免费模式下不可持续 | PDF模板生成+手动签署上传 |
| 招聘管理 | 小企业招人靠人脉和58同城/BOSS直聘，不需要内置招聘 | V3.0考虑 |
| 绩效考核/OKR | 小企业管理扁平化，绩效多靠老板主观判断，不需要系统化 | 不在规划内 |
| 培训管理 | 10-50人企业几乎没有系统化培训需求 | 不在规划内 |
| 社保在线办理(政务接口) | 各地政务接口不统一，对接成本巨大，V1.0阶段ROI极低 | 生成PDF材料供线下办理 |
| 发票OCR识别 | 技术复杂度+API成本，V1.0手动录入足够 | V2.0加入 |
| 个税一键申报 | 需对接自然人电子税务局，认证和接口成本高 | V1.0生成申报表手动提交 |
| 组织架构/部门管理 | 10-50人企业组织简单，通常只有老板+员工两层 | 简化为岗位/角色标签 |
| 审批工作流引擎 | 小企业不需要复杂审批流，老板直接决定 | 固定流程：提交->老板审批，不搞自定义工作流 |
| BI报表/数据大屏 | 小企业老板不需要看复杂图表，当前信息量不值得做BI | V2.0基础统计，不做数据可视化 |
| 深色模式 | 非核心，投入开发资源不值得 | 后续迭代 |
| 多语言/国际化 | 目标用户100%国内中文用户 | 仅中文 |
| 消息通知平台(短信) | 短信成本高，V1.0用APP内消息+微信提醒替代 | 仅APP内推送和微信服务号模板消息 |

### 永久排除

| Anti-Feature | Why Avoid Permanently |
|--------------|----------------------|
| CRM/销售管理 | 超出HR范畴，会让产品失去焦点 |
| 项目管理 | 与人事管理无关，钉钉/飞书已覆盖 |
| 即时通讯/IM | 微信已是不可替代的通讯工具，重复建设无意义 |
| OA审批(请假/出差/加班) | V1.0非核心，小企业靠口头/微信沟通；若V2.0加入也只是简单审批 |
| 员工自助服务门户 | 10-50人企业用微信小程序已足够，不需要独立门户 |

## Feature Dependencies

功能间的依赖关系决定了实现顺序。

```
企业信息设置 ──────────────────────────────────────────────────┐
    |                                                          |
    v                                                          |
员工入职登记 ──→ 员工档案管理 ──→ 薪资结构配置                  |
    |                    |                 |                    |
    v                    v                 v                    |
劳动合同管理        社保参保办理       工资核算 ──→ 电子工资单   |
                         |                 |           |        |
                         v                 v           v        |
                    社保变更联动      个税自动计算   工资发放记录|
                    社保缴费提醒          |                    |
                         |               v                    |
                         v          个税申报表生成            |
                    员工离职办理 ←───────┘                    |
                         |                                    |
                         v                                    |
                    社保停缴提醒 ────────────────────────────┘
```

```
财务模块依赖链：

科目管理 ──→ 凭证管理 ──→ 账簿查询 ──→ 财务报表
                ↑                |
         发票管理               v
                ↑          会计期间管理(结账)
         费用报销 ──→ 审批通过自动生成凭证
              ↑
         员工微信小程序提交

税务计算 ←── 发票管理 + 凭证管理
```

### 关键依赖说明

| Dependency | Description |
|------------|-------------|
| 企业信息 → 所有模块 | 企业城市决定社保政策，统一社会信用代码用于申报 |
| 员工入职 → 社保/工资/个税 | 没有员工档案，后续功能无法运作 |
| 薪资结构 → 工资核算 → 个税计算 | 薪资数据是个税计算的输入源 |
| 工资变动 → 社保基数调整 → 个税重新计算 | 全流程联动是核心价值 |
| 科目管理 → 凭证 → 账簿 → 报表 | 财务标准链路，不可跳跃 |
| 会计期间 → 报表生成 | 结账触发报表，反结账需解锁 |
| 费用报销 → 凭证自动生成 | HR与财务的桥梁功能 |

## MVP Recommendation

### V1.0 必须交付 (MVP Core)

按优先级排序，确保核心流程闭环：

**第一优先级 -- 人事基础闭环（员工全生命周期）：**
1. 企业信息设置 + 手机号登录
2. 员工入职登记（3步完成）
3. 员工档案管理
4. 劳动合同管理（PDF模板）
5. 员工离职办理

**第二优先级 -- 月度运营闭环（老板每月必做）：**
6. 社保参保办理（30+城市政策库）
7. 社保缴费提醒 + 变更联动
8. 薪资结构配置 + 工资核算
9. 电子工资单推送
10. 个税自动计算 + 申报表生成
11. 个税申报提醒

**第三优先级 -- 财务闭环（一站式差异化）：**
12. 会计科目管理（预置小微企业科目）
13. 凭证管理（手动录入+借贷平衡校验）
14. 发票登记（手动）
15. 费用报销（员工提交+老板审批）
16. 账簿查询（总账/明细账/科目余额表）
17. 会计期间管理 + 月末结账
18. 财务报表（资产负债表/利润表）
19. 税务计算辅助

**第四优先级 -- 辅助功能：**
20. 多子账号权限管理
21. 操作日志
22. 员工微信小程序（查看工资条/合同/社保 + 提交报销）
23. 导出功能（Excel/工资条/社保材料/个税申报表）

### Defer 明确延后

| Feature | Reason for Deferral |
|---------|-------------------|
| 考勤管理 | 硬件对接复杂，小企业需求弱，V2.0 |
| 电子签API | 成本问题，V2.0 |
| 发票OCR | 技术复杂度，V2.0 |
| 个税一键申报 | 政务接口复杂，V2.0 |
| 社保第三方数据服务 | 对接成本，V2.0 |
| 人事报表/数据分析 | 数据量不足支撑，V2.0 |
| 招聘模块 | 非核心，V3.0 |
| 付费增值服务 | 先免费获客，V3.0 |

### 最复杂功能 Top 5

1. **社保基数自动匹配** -- 需要维护30+城市政策库，每个城市五险一金基数/比例不同，每年调整
2. **个税自动计算** -- 累计预扣法 + 专项附加扣除匹配 + 年度汇算清缴，算法复杂
3. **财务报表生成** -- 资产负债表/利润表需要准确归集科目数据，涉及会计准则
4. **工资核算** -- 自定义薪资结构 + 扣款项 + 社保个人部分 + 个税 = 最终实发
5. **会计期间管理** -- 结账锁定 + 反结账 + 报表快照 + 凭证修改联动

## Regulatory Compliance Features

中国HR产品必须考虑的合规功能（Table Stakes for compliance）：

| Compliance Area | Feature Requirement | Law/Regulation |
|-----------------|-------------------|----------------|
| 个人信息保护 | 敏感字段加密存储(AES-256-GCM)，数据最小化收集 | 《个人信息保护法》2021 |
| 劳动合同管理 | 合同签订/续签/解除记录保存 | 《劳动合同法》 |
| 社保缴纳记录 | 缴费明细保存，支持打印凭证 | 《社会保险法》 |
| 个税代扣代缴 | 工资个税准确计算，申报记录保存 | 《个人所得税法》 |
| 会计档案 | 凭证/账簿保存，支持导出和打印 | 《会计法》《会计档案管理办法》 |
| 数据安全 | 操作日志可追溯，权限分级控制 | 《数据安全法》 |
