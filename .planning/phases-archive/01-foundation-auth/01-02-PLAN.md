    1. 创建 `pkg/jwt/jwt.go` per D-06/D-27/D-28/D-31：

       定义 Claims 结构体：
       ```go
       type Claims struct {
           UserID int64  `json:"user_id"`
           OrgID  int64  `json:"org_id"`
           Role   string `json:"role"`
           jwt.RegisteredClaims
       }
       ```

       实现 `GenerateAccessToken(userID, orgID int64, role, secret string, ttl time.Duration) (string, error)`：
       - Claims 包含 user_id, org_id, role, exp, iat, jti(UUID)
       - 使用 HS256 签名 per D-06
       - TTL 默认 168h (7天)

       实现 `GenerateRefreshToken(userID int64, secret string, ttl time.Duration) (string, error)`：
       - Claims 仅包含 user_id, exp, iat, jti(UUID)
       - TTL 默认 720h (30天)
       - jti 用于黑名单标识 per D-28

       实现 `ParseToken(tokenStr, secret string) (*Claims, error)`：
       - 解析并验证签名、过期时间
       - 返回 Claims 指针

       实现 `RefreshTokens(refreshTokenStr, secret string, accessTTL, refreshTTL time.Duration, blacklistFunc func(jti string, ttl time.Duration) error) (*TokenPair, error)`：
       - 解析旧 refresh token
       - 调用 blacklistFunc 将旧 jti 加入 Redis 黑名单（per D-28 轮转策略）
       - 生成新 access token + 新 refresh token
       - 返回 TokenPair{AccessToken, RefreshToken}

       定义 `TokenPair` 结构体：`{AccessToken string, RefreshToken string}`

    2. 创建 `pkg/jwt/jwt_test.go`：测试上述所有行为。使用 `github.com/stretchr/testify` 断言。blacklistFunc 用 mock 函数验证调用参数。

       - 调用 `jwt.ParseToken` 解析
       - 检查 Redis 黑名单：key `token:blacklist:{jti}`，如果存在则返回 401
       - 设置 `c.Set("user_id", claims.UserID)`, `c.Set("org_id", claims.OrgID)`, `c.Set("role", claims.Role)`
       - 调用 `c.Next()`
