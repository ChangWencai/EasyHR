## Phase Details

### Phase 1: 登录页 + 布局基础

**Goal:** 用户可以访问登录页并使用完整的应用布局框架
**Depends on:** Nothing
**Requirements:** UI-01, UI-13
**Success Criteria** (what must be TRUE):
  1. 登录页以左右分栏布局正常显示，左侧蓝紫渐变背景+品牌区，右侧白色表单卡片
  2. 登录页渐变背景色（#1A2D6B → #4F6EF7 → #7B9FFF）正确渲染
  3. 登录表单手机号输入框和验证码输入框正常显示，发送验证码按钮显示60s倒计时
  4. AppLayout 侧边栏固定220px宽度正常显示，Logo+菜单项渲染正确
  5. 侧边栏折叠/展开有过渡动画，移动端以抽屉式导航呈现
**Plans:** TBD
**UI hint:** yes

### Phase 2: 首页仪表盘 + 员工管理

**Goal:** 用户可以在首页查看业务仪表盘，在员工管理模块完成员工的查看和新增
**Depends on:** Phase 1
**Requirements:** UI-02, UI-03, UI-04, UI-05
**Success Criteria** (what must be TRUE):
  1. 首页仪表盘显示4个统计卡片（在职员工/本月入职/本月离职/待处理）
  2. 首页图表卡片和待办事项卡片正确渲染，数据来自现有API
  3. 员工列表页筛选栏+表格卡片正常显示，分页器工作正常
  4. 员工详情页左侧信息卡+右侧Tab详情面板正常显示
  5. 新增员工表单显示3步骤条引导，表单卡片居中渲染，步骤切换动画流畅
**Plans:** TBD
**UI hint:** yes

### Phase 3: 薪资管理 + 社保管理

**Goal:** 用户可以在薪资模块和社保模块完成完整的业务操作
**Depends on:** Phase 2
**Requirements:** UI-06, UI-07, UI-08, UI-09
**Success Criteria** (what must be TRUE):
  1. 薪资管理页汇总行显示4个金额统计卡，工资表列表卡片正常渲染
  2. 薪资配置页双栏布局正确（左栏薪资项目+右栏社保/个税配置）
  3. 薪资明细页顶部月份导航器工作，底部实发金额渐变卡片正确显示
  4. 社保管理页顶部橙色警告横幅正常显示，点击可跳转
  5. 社保管理页员工参保状态列表和多行布局正确呈现

### Phase 4: 考勤 + 审批 + 补充页面

**Goal:** 用户可以完成考勤查看、审批处理，以及访问原型未覆盖的补充页面
**Depends on:** Phase 3
**Requirements:** UI-10, UI-11, UI-12, UI-14
**Success Criteria** (what must be TRUE):
  1. 考勤管理页统计行卡片（应出勤/实际出勤/迟到/早退/缺勤）正确显示
  2. 考勤日历表格正常渲染，日期单元格含考勤状态色块
  3. 审批管理页Tab切换正常工作，审批列表卡片状态徽章正确
  4. 审批详情页状态徽章+详情卡正常显示，通过/驳回按钮可用
  5. 工具首页、个人中心、入离职管理等补充页面按原型风格统一设计
**Plans:** TBD
**UI hint:** yes
