## 二、产品概述
### 2.1 产品背景
当前小微企业/个体户老板普遍缺乏专职HR，需自行处理员工入职、社保、工资、个税等人事事务，操作繁琐且易出错。现有工具（钉钉、飞书等）以工作协同为核心，人事功能非专门化、操作复杂，无法满足小老板"简单、好用、省时间"的核心需求。市场暂无专门针对小微企业的轻量化人事管理APP，存在明确市场空白。

### 2.2 目标用户
**核心用户**：小微企业/个体户老板（10-50人规模企业），特征：无专业人事知识、时间精力有限、追求高效便捷、对成本敏感。
**次要用户**：小微企业兼职人事、行政，特征：需快速完成基础人事操作、辅助老板处理事务。

### 2.3 用户核心痛点
1.  社保、个税、工资核算等专业操作复杂，缺乏专业知识易出错。
2.  现有工具功能冗余，操作步骤繁琐，学习成本高。
3.  人事事务分散，需切换多个平台/工具处理，效率低下。
4.  缺乏针对性服务，遇到问题无便捷解决渠道。

## 三、核心功能规划（V1.0版本）
### 3.1 核心功能模块总览
聚焦小微企业每月必做基础人事事务，打造"入职办理-社保管理-工资核算-个税申报-财务记账"全流程闭环，所有功能**前期完全免费**。

> **V1.0 范围说明**：包含五大核心模块（员工入职/离职管理、社保管理、工资管理、个税申报、财务代理记账）+ 基础设置模块。其中财务代理记账模块为 V1.0 基础功能（手动录入凭证/发票登记/费用报销/账簿查询/财务报表），OCR 识别、一键申报等高级功能延后至 V2.0。

### 3.2 功能详情
#### 3.2.1 员工入职/离职管理模块
|功能点|描述|
| ---- | ---- |
|入职登记|老板一键创建入职邀请链接，员工在线填写基本信息、学历、岗位等资料，自动同步至员工档案；支持上传电子劳动合同，在线签署（V1.0 降级方案：PDF 模板生成 + 手动签署上传；V2.0 对接电子签 API 自动签署）。|
|离职办理|员工提交离职申请后，老板在线审核，自动生成离职交接清单（含资产、工作内容）；完成交接后，自动更新员工状态，归档离职资料。|
|员工档案管理|集中存储员工基础信息、合同、入职/离职记录，支持按姓名、岗位快速检索，支持导出Excel档案。|

#### 3.2.2 社保管理模块
|功能点|描述|
| ---- | ---- |
|社保参保办理|根据员工所在城市、岗位，自动匹配社保参保基数（V1.0 采用自建社保政策库，预置 30+ 主流城市数据，管理员后台手动更新；V2.0 对接第三方社保数据服务商）；老板一键提交参保申请，自动生成参保材料（V1.0 生成 PDF 材料供老板线下办理；V2.0 对接政务接口）。|
|社保缴费提醒|提前3天提醒老板完成社保缴费，记录缴费明细，支持查询社保缴纳状态、打印缴费凭证。|
|社保变更办理|员工岗位/薪资变动时，自动触发社保基数调整提醒，老板一键确认完成变更；记录变更历史，可追溯。|

#### 3.2.3 工资管理模块
|功能点|描述|
| ---- | ---- |
|工资核算|支持自定义薪资结构（基本工资、绩效、补贴、扣款等），自动关联员工考勤数据（后续迭代对接），一键核算工资；支持导入考勤表辅助核算。|
|电子工资单|自动生成电子工资单，一键发送至员工手机，员工在线确认签收；支持导出工资条Excel，方便财务对账。|
|工资发放记录|记录每月工资发放状态、金额、发放方式，支持追溯发放历史，异常发放自动提醒。|

#### 3.2.4 个税申报模块
|功能点|描述|
| ---- | ---- |
|个税自动计算|基于工资核算数据，自动匹配个税专项附加扣除项（子女教育、房贷等），精准计算个税金额。|
|个税申报提醒|每月个税申报截止前3天提醒老板，自动生成个税申报表（V1.0 生成申报表供老板手动提交至自然人电子税务局；V2.0 对接个税申报平台一键提交）。|
|个税记录查询|记录每月个税申报明细，支持查询申报状态、更正记录，导出个税申报凭证。|

#### 3.2.5 财务代理记账模块
|功能点|描述|
| ---- | ---- |
|凭证管理|支持手动录入会计凭证（选择科目、填写借贷金额、摘要），支持凭证草稿保存、提交审核、审核通过后自动更新科目余额；凭证按月度编号，支持按期间/科目/摘要搜索。|
|发票管理|支持进项/销项发票登记（手动录入或拍照识别，V1.0 手动录入为主；V2.0 对接税务局发票查验接口自动验真 + OCR 识别）；发票关联至凭证，月末汇总自动计算增值税进项/销项税额。|
|费用报销|员工通过微信小程序提交费用报销单（上传票据照片、填写金额/类型），老板在线审批（通过/驳回），审批通过后自动生成费用凭证；支持报销状态追踪（待审批/已通过/已支付）。|
|账簿查询|基于凭证数据实时自动生成总账、明细账、科目余额表，支持按科目、按期间查询，支持导出Excel。|
|财务报表|月末结账后自动生成资产负债表、利润表；报表数据按结账时点快照存储，后续凭证修改不影响已生成报表；支持多期对比分析。|
|会计期间管理|支持按月度开启/关闭会计期间，结账时锁定当期凭证不可修改；支持反结账（需二次确认），结账后自动触发报表生成。|
|科目管理|预置小微企业常用会计科目（简化版），支持企业在标准科目基础上自定义增删科目；科目按资产/负债/权益/成本/损益五大类分层展示。|
|税务计算|基于发票和凭证数据自动计算增值税（销项-进项）、企业所得税（季度预缴）；生成纳税申报辅助数据，导出申报表（V1.0 导出 Excel 辅助申报；V2.0 对接税务系统一键申报）。|

#### 3.2.6 基础设置模块
|功能点|描述|
| ---- | ---- |
|企业信息设置|录入企业名称、统一社会信用代码、所在城市等基础信息，同步至各功能模块，减少重复填写。|
|员工权限设置|支持添加多个子账号（如兼职人事），分配不同操作权限（如仅查看、仅办理入职）；主账号拥有全部权限。|
|消息提醒设置|自定义各功能模块提醒时间、方式（短信、APP内消息、微信），满足个性化需求。|

## 四、非功能需求
### 4.1 性能需求
1.  响应时间：APP端操作响应≤2秒，接口调用响应≤1秒。
2.  并发能力：支持同时在线用户≥1000，单企业同时操作≤50人时无卡顿。
3.  稳定性：全年可用性≥99.9%，支持7×24小时稳定运行。

### 4.2 兼容性需求
1.  系统适配：支持Android 8.0及以上、iOS 12.0及以上系统。
2.  终端适配：适配手机、平板等主流设备，保证界面显示正常、操作流畅。

### 4.3 安全需求
1.  数据安全：员工个人信息、企业数据加密存储，防止数据泄露；支持数据备份与恢复。
2.  权限安全：严格控制子账号权限，避免越权操作；操作日志全程记录，可追溯。
3.  合规性：符合《个人信息保护法》《劳动合同法》等相关法律法规要求。

### 4.4 易用性需求
1.  操作流程：核心功能操作步骤≤3步，无需专业知识即可完成。
2.  界面设计：简洁清晰，无冗余信息，重点功能突出，适配小老板操作习惯。
3.  引导支持：首次使用提供新手引导，关键操作提供弹窗提示/帮助文档。

### 4.5 验收标准
| 需求项 | 验收标准 | 测试方法 |
|--------|---------|---------|
| 响应时间 | 95% 请求 ≤2s | JMeter 压测，模拟 50 并发连续操作 |
| 并发能力 | 1000 用户在线无崩溃 | 负载测试，持续 30 分钟 |
| 操作步骤 | 核心功能 ≤3 步完成 | 可用性测试，5 名目标用户实操 |
| 数据安全 | 敏感字段 AES-256 加密存储 | 数据库直接查询验证 |
| 权限控制 | 子账号无法越权操作 | 渗透测试，遍历所有角色权限矩阵 |

## 五、产品迭代规划
### 5.1 V1.0（首发版本）
完成核心五大模块：入职/离职管理、社保管理、工资管理、个税申报、财务代理记账（基础功能），基础功能免费，积累种子用户。
- 电子签：PDF 模板生成 + 手动签署上传（降级方案）
- 社保政策：自建政策库，30+ 城市，手动更新
- 个税申报：自动计算 + 手动提交至自然人电子税务局
- 发票管理：手动录入 + 拍照识别（OCR 延后）

### 5.2 V2.0（3个月后迭代）
- 新增考勤管理模块（对接打卡设备/手机定位打卡）
- 员工合同到期提醒、人事报表自动生成（企业人事数据统计分析）
- 电子签 API 自动签署（对接上上签/e签宝）
- 社保政策对接第三方数据服务商
- 发票 OCR 识别 + 税务局查验接口对接
- 个税一键申报对接

### 5.3 V3.0（6个月后迭代）
- 新增招聘模块（对接本地招聘渠道，一键发布招聘信息）
- 付费增值服务（人事合规咨询、社保/个税代办服务）
- 社保在线办理对接政务接口

## 六、商业模式
### 6.1 前期（V1.0-V2.0）
核心功能**完全免费**，通过免费功能获取用户，沉淀企业与员工数据。

### 6.2 中期（V3.0及以后）
1.  增值服务收费：人事合规咨询、社保/个税代办、定制化报表等付费服务。
2.  功能模块付费：进阶考勤、招聘、绩效等模块按年/按月付费。
3.  数据服务：基于沉淀数据输出行业薪酬报告，向企业/机构收费。

## 七、风险与应对
|风险点|应对措施|
| ---- | ---- |
|社保政策变动导致功能适配不及时|V1.0 采用自建政策库 + 管理员手动更新，政策变更时自动标记受影响员工并推送确认提醒；V2.0 对接第三方社保数据服务商，降低维护成本。|
|用户接受度低，推广困难|聚焦核心痛点，打造极致简化的操作体验；通过小老板社群、线下地推等精准渠道获客，收集用户反馈快速优化。|
|数据安全风险|加强技术加密与权限管控，定期开展安全审计；建立数据泄露应急处理方案，降低风险损失。|
|第三方接口不可用|所有第三方依赖均设计降级方案（电子签→手动签署、社保→手动录入、个税→手动申报），核心业务不依赖外部接口可用性。|
